HTML Code-Exploration

What is HTML?

HTML stands for HyperText Markup Language and is used to create the structure and content of a webpage.

Boilerplate code

It refers to repetitive code segments that are often used without modification in multiple programming projects. These code snippets typically serve as a foundation for various tasks and can be reused to save time and effort.

What are Tags?

HTML tags are like keywords that tell the web browser how to format and display content on a web page. They are enclosed in angle brackets (< and >), and they usually come in pairs [opening tag (<p>) and closing tag (</p>)]. There are also few self closing tags.

What are Elements?

An HTML element is a building block of an HTML document. It defines a specific part of the content, such as a heading, paragraph, image, or link. Each element has a specific structure, often consisting of an opening tag, content, and a closing tag (<h1>content</h1>).

What are Attributes?

An HTML attribute is a special word used inside the ,opening tag of an HTML element to control its behavior or appearance. It provides additional information about the element, such as its properties, values, or relationships to other elements.

Link and Media Tags

Anchor tag

It creates a hyperlink to a target URL. It's essentially used to link one web page to another. When a user clicks on the link created by the anchor tag, they are taken to the specified destination.

<a href="https://example.com" target="_blank">Visit example.com</a>

Attributes:

  1. href:Specifies the URL that the link will point to.
  2. target: Controls where the linked document will be opened.
  3. target=_blank: Opens the link in a new tab or window.
  4. target=_self: Opens the link in the same window (default).
  5. target=_top: Replaces the entire content of the window with the linked document.
  6. target=_parent: Opens the link in the parent frame of the current frame.

Img tag

The <img> tag is used to embed an image into an HTML document. It allows you to display images on a webpage.

<img src="Source_of_image" alt="Alternate text" width="400" height="300">

Attributes:

  1. src:Specifies the URL of the image file.
  2. alt:Specifies an alternative text description for the image.
  3. width:Specifies the width of the image in pixels.
  4. heigth:Specifies the height of the image in pixels.

Audio tag

The <audio> tag is used to embed an audio file into an HTML document. It allows you to play audio directly on a webpage.

<audio src="audio.mp3" controls autoplay loop muted preload="auto"> 
Your browser does not support the audio element.</audio>

Attributes:

    The text "Your browser does not support the audio element" is displayed if the browser cannot play the audio.
  1. src:Specifies the URL of the audio file.
  2. controls: Adds playback controls like play, pause, volume, etc. to the audio player.
  3. autoplay: Automatically starts playing the audio when the page loads.
  4. loop: Repeats the audio after it finishes playing.
  5. muted: Mutes the audio by default.
  6. preload="none"No preloading occurs (default).
  7. preload="metadata" Only the metadata (like duration, title, etc.) of the audio file is preloaded. This is the default value.
  8. preload="auto" The entire audio file is preloaded.

Video tag

The <video> tag is used to embed a video into an HTML document. It allows you to play videos directly on a webpage.

<video src="video.mp4" width="340" height="255" controls autoplay loop muted poster="thumbnail.jpg"> 
Your browser does not support the video tag.</video>

Attributes:

  1. src: Specifies the URL of the video file.
  2. controls: Adds playback controls like play, pause, volume, etc. to the video player.
  3. autoplay: Automatically starts playing the video when the page loads.
  4. loop: Repeats the video after it finishes playing.
  5. muted: Mutes the video by default.
  6. width: Specifies the width of the video player.
  7. height: Specifies the height of the video player.
  8. poster: Specify an image that will be displayed before the video starts playing (Thumbnail).

Forms

A form is an HTML element used to collect user input data. It creates an interactive interface where users can enter information, select options, or submit data to a server.

<form> action="/action.php" method="post"></form>

Attributes:

  1. action: Specifies the URL of the page that will process the form data when it is submitted.
  2. method: Specifies the HTTP method (GET or POST) used to send the form data to the server.
  3. name: Specifies a name for the form (optional).
  4. id: Specifies a unique ID for the form (optional).

Input

An input element is a fundamental component of HTML forms that allows users to enter data into a webpage.

<input type="text" name="username" value="John" placeholder="Enter username">

Attributes:

  1. type: Specifies the type of input field.
  2. name: Specifies the name of the input field.
  3. value: Sets the initial value of the input field.
  4. placeholder: Provides a hint to the user about the expected input.

Label

A label element in a form is used to associate a text label with a form control, such as an input field, select element, or textarea.

<label for="username">Username:</label>

Attributes:

  1. for: Specifies the ID of the form control that the label is associated with.
  2. accesskey: Specifies a shortcut key that the user can press to focus on the associated form control.

Password

a password field is a text input that is used to collect sensitive information, such as a user's password. When a user enters text into a password field, the characters are typically masked (usually with asterisks or dots) to prevent others from seeing them.

<input type="password" name="password" placeholder="Enter password">

Textarea

A textarea element in a form is used to create a multi-line text input field. It allows users to enter larger amounts of text, such as paragraphs or essays.

<textarea name="description" rows="5" cols="30">Enter description</textarea>

Attributes:

  1. rows: Specifies the number of rows in the textarea.
  2. cols: Specifies the number of columns in the textarea.
  3. name: Specifies a name for the textarea (optional).
  4. id: Specifies a unique ID for the textarea (optional).
  5. placeholder: Specifies a hint text to be displayed in the textarea before the user enters any text.

Checkbox

A checkbox is a type of input element in a form that allows the user to select one or more options from a list. It typically appears as a small square that can be checked or unchecked.

<input type="checkbox" name="agree" value="yes"> I agree</input>

Attributes:

  1. name: Specifies the name of the checkbox, which is used to identify the value when the form is submitted.
  2. value: Specifies the value that will be submitted to the server when the checkbox is checked. If the checkbox is unchecked, no value is submitted.
  3. checked: Indicates that the checkbox is checked by default.
  4. disabled: Disables the checkbox, preventing the user from checking or unchecking it.

Button

A button in a form is an interactive element that allows users to trigger specific actions or submit form data. It typically appears as a clickable button with a label or icon.

<button type="button">Click Me</button>

Attributes:

  1. type: Specifies the type of button. Common values include:
  2. value: Specifies the value of the button (e.g., the text that appears on the button).
  3. disabled: Disables the button, making it unclickable.
  4. name: Specifies a name for the button (optional).
  5. id: Specifies a unique ID for the button (optional).

Radio button

A radio button is a type of form control that allows the user to select one option from a group of mutually exclusive options. When one radio button is selected, all other radio buttons in the same group are automatically deselected.

<input type="radio" name="gender" value="male"> Male>
<input type="radio" name="gender" value="female"> Female>

Attributes:

  1. name: Specifies the name of the radio button group. All radio buttons within the same group must have the same name.
  2. value: Specifies the value that will be submitted to the server when the radio button is selected.
  3. checked: Indicates that the radio button is selected by default.

Submit button

A submit button is a type of button element used in forms to send the form data to the server for processing. When the user clicks on the submit button, the form's action is triggered, and the collected data is sent to the specified URL.

<input type="submit" value="Submit"></input>

Attributes:

  1. value: Specifies the text that appears on the button.
  2. name: Specifies a name for the button (optional).
  3. disabled: Disables the button, preventing it from being clicked.

    Form Validation

    Required attribute

    This attribute specifies that the field must have a value before the form can be submitted.

    <form>
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required>
  <br>
  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>
  <br>
  <label for="password">Password:</label>
  <input type="password" id="password" name="password" required pattern=".{8,}" title="Password must be at least 8 characters">
  <br>
  <input type="submit" value="Submit">
</form>

    The name and email fields are required.The password field is required and must be at least 8 characters long.

    Pattern

    This attribute defines a regex(regular expression) that the value must match.

    <label for="password">Password:</label>
<input type="password" id="password" name="password" required pattern=".{8,}" title="Password must be at least 8 characters">

    The password field must contain at least 8 characters.

    Min attribute

    Specifies the minimum value allowed for the input.

    <label for="age">Age:</label>
<input type="number" id="age" name="age" min="13" required>

    min="13": Sets the minimum allowed value to 13. It ensure that your website is only accessible to users who meet the minimum age requirement.

    Max attribute

    Specifies the maximum value allowed for the input.

    <label for="age">Age:</label>
<input type="number" id="age" name="age" min="0" max="120">

    The max attribute is set to 120 for the age input field. This means that the user cannot enter a value greater than 120. If they do, the form will not be submitted, and an error message will be displayed.

    Minlength attribute

    Specifies the minimum number of characters allowed in the input field.

    <label for="phone">Phone number:</label>
<input type="tel" id="phone" name="phone" minlength="10">

    The minlength attribute is set to 10 This means that the user must enter at least 10 characters.

    Maxlength attribute

    Specifies the maximum number of characters allowed in the input field.

    <label for="phone">Phone number:</label>
<input type="tel" id="phone" name="phone" minlength="10" maxlength="15">

    The minlength attribute is set to 10 and the maxlength attribute is set to 15. This means that the user must enter at least 10 characters but no more than 15 characters.

Miscellaneous

Entities

HTML entities are special sequences of characters that represent characters that are difficult to represent directly in HTML, such as special characters, characters that have special meaning in HTML, or characters that are not supported by the character encoding being used.

&amp;: Represents the ampersand character (&)
&lt;: Represents the less than sign (<)
&gt;: Represents the greater than sign (>)
&nbsp;: Represents a non-breaking space
&copy;: Represents a copyright symbol (©)
Note: add & infront of entities.

Linking

  2. Set the rel attribute to "stylesheet" to indicate that the linked file is a CSS stylesheet.
  3. Set the href attribute to the path of the CSS file.

Linking js

<script src="script.js">

  1. Add a <script> tag to the <head> or <body> section of your HTML document.
  2. Set the src attribute to the path of the JavaScript file.
